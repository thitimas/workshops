1 Machine Learning Basics

We focus on supervised learning — the model learns from labeled examples and makes predictions on new data. The two task types you need to know:

Regression

Continuous Output

Predicts a number on a continuous scale.
Example: Given a house's size and location, predict its price — e.g., $347,200.

Classification

Discrete Output

Assigns input to a category, with a confidence score.
Example: Given an image, predict "cat" with 94% confidence or "dog" with 6%.

Model-Based Learning

The system builds a model from training examples, then uses that model to make predictions on new, unseen inputs. The key challenge is hyperparameter tuning — choosing settings like learning rate and network depth through empirical testing. This is often the most tedious part of the process.

2 Tensors & TensorFlow

A tensor is a way to store numbers in multiple dimensions — it's the fundamental data structure used in machine learning.

StructureExampleDimensionsUse in ML

TensorFlow is a software library built around the idea of data flowing through a network as multi-dimensional tensors — hence the name.

3 Artificial Neural Networks

Feedforward Network

Information flows in one direction: input → hidden layers → output. There are no loops. Every node in each hidden layer is connected to all nodes in the previous layer. This is the architecture our project uses.

Recurrent Network (not our focus)

Has loops — output can feed back into earlier layers. Useful for sequences (text, speech). We won't use this for the poisoning project.

The Perceptron (one neuron)

Inspired by the human brain. Each neuron:

1. Receives inputs from the previous layer, each multiplied by a weight.
2. Sums the weighted inputs.
3. Applies an activation function — if the sum exceeds a threshold, the neuron fires (output = 1); otherwise it doesn't contribute (output ≈ 0).

The weights are the unknowns. Training finds optimal weight values by repeatedly adjusting them to minimize prediction error.

4 Convolutional Neural Networks (CNN)

CNNs are built on top of classical image processing. Understanding filters first makes CNN intuitive.

The Signal Processing Analogy

Think of an image as a 2D signal. Like audio signals, you can decompose it into frequency components (via 2D Fourier transforms):

• Low frequencies = overall structure, shapes, large regions of similar color.
• High frequencies = fine details, sharp edges, noise.

A filter (kernel) is a small matrix that slides across the image, computing a dot product at each position. This sliding-window operation is called convolution — that's where the "Conv" in CNN comes from.

From Filters to CNN Layers

In classical image processing, engineers manually design these filter kernels. In a CNN, the filter values are the weights — they are automatically learned through training, just like the perceptron weights above.

A CNN stacks multiple types of layers:

Layer TypeWhat it does
ConvolutionalApplies learned filters → produces feature maps (e.g., one map per filter detecting edges, curves, etc.)
Activation (ReLU)Applies non-linearity — sets negatives to 0, keeps positives
Pooling (Sub-sampling)Shrinks the feature map (e.g., max pooling takes the largest value in each region) — reduces computation and overfitting
Fully ConnectedFinal layers that combine all learned features and output a class prediction with confidence scores

5 Data Poisoning Attacks

Our project explores how a trained image classifier can be fooled by imperceptibly small modifications to input images.

Open Question

Can we minimize the noise?

If we add noise to an image to trick the model, can we make that noise so small that a human can't even see it? Or does making it harder to detect also make it harder to fool the model?

Open Question

What if we can't see inside the model?

If we don't have access to the model at all (a "black box") — how do we figure out what noise to add so that the model gives the wrong answer?
